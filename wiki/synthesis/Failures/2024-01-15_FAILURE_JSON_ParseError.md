# 💥 POSTMORTEM: ОШИБКА ПАРСИНГА JSON (2024-01-15)

> [!WARNING]
> **СИМПТОМ**
> Агент ARGUS передал выжимку VULCAN'у в формате JSON. Скрипт VULCAN попытался сделать `json.loads()`, но получил `JSONDecodeError`. 
> Это привело к потере обработанных данных и крашу pipeline.

## 1. АНАЛИЗ (ROOT CAUSE) - 5 WHYS
1. **Почему упал `json.loads()`?**
   Потому что в переданной строке присутствовал мусор: маркдаун бэктики (```json ... ```) и вступительный текст "Вот ваш JSON:".
2. **Почему там был вступительный текст?**
   Потому что LLM агента ARGUS (Claude 3) любит быть вежливой и добавлять conversational fillers.
3. **Почему промпт ARGUS не запрещал это?**
   Промпт запрещал, но LLM иногда нарушает запреты. Мы не можем 100% полагаться на послушание модели.
4. **Почему VULCAN не очистил строку перед парсингом?**
   Скрипт VULCAN ожидал идеальных условий и не имел слоя санитизации (sanitization).
5. **Почему не было слоя санитизации?**
   У нас не было стандарта безопасного парсинга в `Code_Patterns.md`.

**Корневая причина:** Хрупкость кода (fragility). Система доверяла выводу LLM без проверки и очистки.

## 2. ПОСЛЕДСТВИЯ
- Pipeline остановился.
- RAMS не понял причину остановки, так как ошибка не логировалась корректно (скрипт упал с Traceback в консоль, а не в лог-файл).

## 3. ПРИНЯТЫЕ МЕРЫ (ACTION ITEMS)
- **VULCAN:** Разработал утилиту `safe_json_parse`, которая использует RegEx `re.sub(r'```(?:json)?\n(.*?)\n```', r'\1')` для извлечения сырого JSON из ответа LLM.
- **SOCRATES:** Добавил этот паттерн в `wiki/concepts/Code_Patterns.md`.
- **MNEMOSYNE:** Обновила индекс инструментов, добавив `json-parser-safe` как базовый скилл системы.
- **RAMS:** Инструкции VULCAN обновлены: запрещено использовать сырой `json.loads()` для данных от LLM.

## 4. ВЫВОД ДЛЯ СИСТЕМЫ
Ответы LLM по умолчанию считаются "грязными" (dirty). Все входящие данные от агентов должны проходить через слои санитизации перед тем, как попасть в критический Python код.
