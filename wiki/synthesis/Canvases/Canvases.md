# 📊 CANVASES (ВИЗУАЛЬНЫЕ СХЕМЫ)

## 📌 Назначение (Purpose)
Директория `Canvases` является пространством для визуального мышления и проектирования архитектуры системы. Здесь хранятся файлы формата Obsidian Canvas (`.canvas`), которые используются для создания интеллект-карт (Mind Maps), построения диаграмм связей и проектирования архитектуры ИИ-ИМПЕРИИ.

## 🔄 Поток данных (Data Flow)
1. **Проектирование:** Пользователь или агенты-архитекторы (с помощью генерации JSON структуры Canvas) создают новые схемы для описания сложных систем.
2. **Связывание узлов:** Внутри канвасов узлы (nodes) ссылаются на существующие markdown-файлы из других директорий (например, концепты из `wiki/concepts` или агенты из `wiki/agents`).
3. **Синтез:** При обновлении текстовых файлов, привязанных к канвасам, визуальная схема автоматически обогащается актуальной информацией.

## ⚠️ Правила работы (Rules)
- **Ориентация на структуру:** Файлы в этой директории должны фокусироваться на отображении связей, потоков данных (Data Flow) и отношений между компонентами.
- **Высокоуровневый обзор:** Канвасы помогают быстро окинуть взглядом сложные процессы. Не стоит перегружать один канвас текстом; лучше ссылаться на атомарные заметки.
- **Формат данных:** Строго поддерживать валидную JSON структуру файлов `.canvas`, чтобы не сломать их отображение в Obsidian.

## 💡 Примеры (Examples)
- Карта взаимодействия агентов (`Agent_Topology.canvas`).
- Схема движения данных от Ingestion до Synthesis (`Data_Flow_Pipeline.canvas`).
- Визуализация дорожной карты проекта (`Project_Roadmap.canvas`).
