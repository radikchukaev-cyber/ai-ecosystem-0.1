# Журнал устранения ошибок

## Описание
Документ для протоколирования процесса решения сложных системных ошибок и багов (Bug Tracking & Resolution).
В отличие от журнала инцидентов (который фокусируется на простоях), этот лог сфокусирован на дефектах в программном обеспечении.
Обеспечивает накопление базы знаний (Knowledge Base) для ускорения решения аналогичных проблем в будущем.

## Роли и ответственность
* **Специалист по тестированию (QA)**: воспроизведение ошибок и создание баг-репортов.
* **Разработчик (Bug Fixer)**: локализация проблемы, написание патча (Fix) и тестов к нему.
* **Технический писатель (Tech Writer)**: документирование обходных путей (Workarounds) для пользователей.

## Архитектура и концепции
Связан с системами трекинга задач (Jira, Bugzilla, GitHub Issues).
Каждая запись включает шаги для воспроизведения (Steps to Reproduce), ожидаемое и фактическое поведение.
Обязательной практикой является добавление регрессионного теста (Regression Test) перед закрытием бага.
