# 🔌 OBSIDIAN PLUGINS

Директория `OBSIDIAN_PLUGINS` служит центральным хабом для документации, настройки и управления жизненным циклом плагинов Obsidian, которые превращают обычный текстовый редактор в мощную среду для работы LLM-агентов (Агентурную IDE). 

### Архитектура Интеграций

В системе "Карпаты" Obsidian выступает не просто как база знаний, а как полноценная база данных, шина сообщений и графический интерфейс. Плагины здесь играют роль системных драйверов:
- **Интерфейсные плагины (UI/UX):** Настройки Dataview, Kanban, Templater, которые структурируют визуальное отображение для Шефа.
- **Системные коннекторы (Connectors):** Плагины для автоматической синхронизации (Git/Sync), обеспечивающие версионирование и бэкап.
- **LLM-Интеграции:** Плагины типа Smart Connections, Text Generator, или локальные коннекторы, через которые агенты (ВУЛКАН, МНЕМОЗИНА) напрямую читают, индексируют и модифицируют графовую структуру заметок.

### Использование и Принципы

1. **Единый стандарт (Source of Truth):** Каждый установленный плагин должен иметь здесь свой конфигурационный файл или описательную заметку, где задокументированы его предназначение, версия и потенциальные конфликты с другими расширениями.
2. **Синхронизация Агентов:** Агент МНЕМОЗИНА использует эти манифесты для понимания структуры метаданных (Frontmatter), которые генерируются плагинами, а агент ВУЛКАН опирается на них при написании скриптов автоматизации.
3. **Ограничение хаоса:** Запрещается установка плагинов, которые нарушают целостность Markdown-стандарта (Lock-in плагины). Приоритет отдается open-source решениям, которые сохраняют читаемость данных в виде простого текста.
4. **Оптимизация производительности:** Здесь фиксируются правила отключения "тяжелых" плагинов на мобильных устройствах или слабых клиентах, чтобы не замедлять индексацию графа.

Этот узел критически важен для поддержки работоспособности всей экосистемы Obsidian как интерфейса для ИИ-Империи.
