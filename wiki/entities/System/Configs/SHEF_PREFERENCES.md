# Предпочтения Шефа (SHEF_PREFERENCES)

## 1. Архитектурные и Технические Стандарты
Шеф установил безупречные строгие правила для развития архитектуры ИИ-Империи.
Всё должно быть минималистично, быстро, надежно и элегантно. Громоздкие фреймворки отвергаются.

## 2. Технологический Стек
- **Языки:** Python (для ИИ и работы с данными), Node.js/TypeScript (для микросервисов), Bash (для системной магии).
- **Инфраструктура:** Docker и Kubernetes для глобальной оркестрации и изоляции агентов.
- **База Знаний:** Чистый Markdown + Obsidian (локально, никаких внешних облаков).

## 3. Требования к UI / UX
- **Тема:** Строго Dark Mode. Любой агент, генерирующий светлые интерфейсы, будет декомпилирован на месте.
- **Шрифты:** Только Monospace (Fira Code, JetBrains Mono) для максимальной читаемости логов.
- **CLI over GUI:** Интерфейсы командной строки всегда в приоритете перед графическими оболочками.

## 4. Концепции Разработки
- **No-Code / Low-Code:** Применяются только для быстрого прототипирования (n8n).
- **AI-Driven Development:** Код должны писать машины, люди только контролируют архитектуру и направляют поток.
- **Zero Trust:** Никакого доверия внешним API по умолчанию, всегда ожидать падения и писать фоллбеки.

## 5. Строгие Системные Лимиты
- Максимальная длина функции: 40 строк кода. Если больше — требуется декомпозиция.
- Максимальное время ответа API: 200ms. Задержки считаются критическим сбоем.
- Покрытие тестами: не менее 85% для Core-модулей Империи.
