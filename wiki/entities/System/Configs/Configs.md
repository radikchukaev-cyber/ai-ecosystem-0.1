# ⚙️ SYSTEM CONFIGS

Эта директория представляет собой централизованное хранилище всех конфигурационных файлов, переменных окружения и профилей нашей интеллектуальной системы. Данный узел выполняет роль "нервной системы", обеспечивая единообразие настроек для всех подсистем, сервисов и LLM-агентов, работающих в рамках инфраструктуры "Карпаты".

### Архитектура и Структура

1. **Глобальные параметры (Environment Variables):** Хранение `.env` шаблонов, базовых URL-ов для внутренних API и путей к моделям.
2. **Профили доступа (Roles & Permissions):** Политики RBAC, определяющие, к каким ресурсам имеет доступ каждый конкретный агент (например, ограничения на запись для `Researcher` или полные права для `VULCAN`).
3. **Настройки среды (Environment Configs):** Конфигурации для Obsidian (темы, хоткеи, сниппеты), настройки N8N (интервалы поллинга, лимиты) и других интеграций.
4. **Профиль Шефа (User Preferences):** Индивидуальные настройки пользователя, его стилевые предпочтения, глоссарий и персональные промпты.

### Принципы использования

- **Безопасность (Security-First):** Любые изменения в данной директории требуют верификации. Секретные ключи не должны храниться в открытом виде, вместо них используются ссылки на vault-хранилища или переменные окружения.
- **Изоляция:** Каждая подсистема должна обращаться только к своей конфигурации, минимизируя зависимости от глобального состояния.
- **Мониторинг:** Регулярно проводится автоматический аудит целостности конфигов для предотвращения случайных сбоев при обновлениях плагинов или системных модулей.
- **Версионирование:** Все изменения в файлах настроек строго отслеживаются через Git, чтобы обеспечить возможность быстрого отката в случае деградации производительности.

Эта директория — фундамент стабильности. Любые агенты, работающие с настройками, обязаны сверяться с этими стандартами перед выполнением конфигурационных мутаций.
