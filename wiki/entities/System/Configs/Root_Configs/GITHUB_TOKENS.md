# Корневая Конфигурация: GITHUB_TOKENS

## 1. Взаимодействие с Репозиториями
GitHub — это главная кузница Империи. Именно здесь алгоритмические сущности превращают стратегические мысли Шефа в работающие боевые механизмы.
Управление токенами доступа критично для фундаментальной безопасности всей кодовой базы.

## 2. Типы Токенов и Их Роли
- **CI/CD Actions Token:** Имеет права исключительно на чтение кода и запуск изолированных тестов.
- **Agent Push Token:** Используется автономными ИИ-кодерами. Разрешения: создание feature-веток, генерация коммитов, открытие Pull Requests.
- **Root Token (Shef-only):** Абсолютный доступ, включая удаление репозиториев, изменение политик защиты веток и управление секретами.

## 3. Жесткие Политики Слияния (Merge Policies)
- Ни одному агенту не разрешено напрямую коммитить в ветки `main` или `master`. Это правило не имеет исключений.
- Только Шеф (или авторизованный Титан-Архитектор) имеет право аппрувить Pull Requests.
- Каждый PR обязан проходить цепочку линтеров, тесты (Test Coverage > 85%) и статический анализ на уязвимости (Dependabot/Snyk/Sonar).

## 4. Системные Лимиты
- Максимальное количество одновременно открытых PR от одного агента: 5. (Защита от спама).
- Срок жизни Agent Token: 7 дней (с обязательной динамической регенерацией).
- В случае попытки Force Push в защищенную ветку (грубая ошибка агента): токен немедленно аннулируется, сам агент подлежит уничтожению и замене на новую версию.
