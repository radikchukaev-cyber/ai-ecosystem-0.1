# ⚙️ ROOT CONFIGS

Директория `Root_Configs` является наиболее защищенным сегментом системной архитектуры, так называемым "Ring 0" нашей файловой системы. Это ядро безопасности, предназначенное для хранения высококритичных конфигураций, которые определяют базовое функционирование инфраструктуры "Карпаты".

### Архитектурный Контекст

Здесь локализованы фундаментальные настройки, которые напрямую влияют на работоспособность всех внешних интеграций и внутренних модулей.
- **Секреты и API Ключи:** Токены доступа к LLM-провайдерам (OpenAI, Anthropic), ключи для GitHub, YouTube API и прочих критически важных сервисов.
- **Управление доступом (RBAC):** Глубинные настройки ролевой модели, определяющие границы допустимого для каждого автономного агента.
- **Системные Триггеры:** Конфигурации корневых N8N-воркфлоу, оркеструющих жизненный цикл системы на макро-уровне.
- **Защищенные профили:** Базовые настройки системных демонов и сервисов, отвечающих за резервное копирование и синхронизацию.

### Правила и Ограничения Использования

1. **Строгий контроль доступа (Zero Trust):** К этой директории имеют доступ исключительно системный администратор (Шеф) и оркестратор высшего уровня (RAMS). Рядовые агенты (например, аналитики или сборщики данных) физически не могут читать эти файлы.
2. **Аудит мутаций:** Любая попытка изменить содержимое этих конфигураций логируется в системном журнале. Изменения должны быть явно подтверждены человеком.
3. **Безопасное извлечение (Safe Retrieval):** При необходимости использования ключей, они загружаются в память только в момент исполнения защищенного процесса и не передаются в текстовом виде в промпты или открытые логи.
4. **Резервирование:** Конфигурации данного уровня подлежат регулярному зашифрованному бэкапу на независимые носители.

`Root_Configs` гарантирует, что даже в случае компрометации отдельного агента или плагина, вся архитектура останется защищенной и управляемой.
