# Корневая Конфигурация: API_KEYS

## 1. Концепция Управления Секретами
API ключи — это драгоценная кровь Империи, позволяющая агентам взаимодействовать с внешними закрытыми мирами.
Шеф категорически запрещает хардкодинг ключей в репозиториях, скриптах или базах данных в открытом виде.

## 2. Архитектура Защищенного Хранения
Все критические ключи (OpenAI, Anthropic, AWS, Stripe и др.) хранятся в защищенном изоляторе (Vault).
Агенты получают временные ключи динамически исключительно на время выполнения конкретной задачи (TTL = 1 hour).
- **Environment Policy:** Только `production` и `sandbox` контуры.
- **Encryption Standard:** AES-256-GCM в состоянии покоя.

## 3. Роли и Уровни Доступа к Ключам
- **Titan Agents:** Привилегированный доступ к дорогим ключам GPT-4, Claude Opus.
- **Worker Agents:** Выделяются только локальные LLM (без ключей) или дешевые API-модели для массовых задач.
- **Scraper Agents:** Используют отдельный пул ключей для ротации прокси и решения капчи.

## 4. Строгие Системные Лимиты и Наказания
- При малейшем обнаружении ключа в публичном или приватном коммите: немедленный автоматический отзыв ключа (Revoke), блокировка агента-виновника, запуск протокола агрессивной очистки истории Git.
- Финансовый лимит на один выданный ключ: $10 в сутки (Hard Cap), предотвращающий утечку средств.
- Все внешние ключи обязаны ротироваться каждые 90 дней автоматически без ручного вмешательства.
