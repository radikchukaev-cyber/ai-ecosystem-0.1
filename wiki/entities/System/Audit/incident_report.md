# Отчет об инцидентах

## Описание
В этом документе содержатся подробные отчеты о произошедших ИТ-инцидентах (Incident Reports / Post-Mortems).
Описываются причины сбоя, хронология событий, влияние на бизнес и предпринятые меры для предотвращения в будущем.
Отчеты об инцидентах являются ключевым инструментом для непрерывного улучшения процессов.

## Роли и ответственность
* **Менеджер инцидентов (Incident Manager)**: расследование и составление итогового отчета.
* **Команда разработки (R&D)**: техническая экспертиза корневых причин (Root Cause Analysis).
* **Служба поддержки**: предоставление таймлайна обращений пользователей.

## Архитектура и концепции
Применяется методология "Blameless Post-Mortem" (отчеты без поиска виноватых, фокус на системных проблемах).
Используются методы "Пять почему" (5 Whys) и диаграммы Исикавы (Fishbone) для глубокого анализа.
Все извлеченные уроки транслируются в задачи для беклога разработки (Action Items).
