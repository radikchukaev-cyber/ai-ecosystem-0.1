# 🧪 SYSTEM TESTS

## Общее назначение
Директория `System/Tests` служит центральным репозиторием для всех юнит-тестов, интеграционных тестов и E2E (End-to-End) сценариев, которые покрывают системные скрипты и автоматизации комплекса. Тестирование является краеугольным камнем стабильности, предотвращая регрессии при обновлении функционала агентами.

## Архитектурный контекст
Агент **VULCAN** (главный инженер и кодер) использует эту директорию как свой полигон перед любым слиянием кода или деплоем в production-среду системы. Архитектура тестов разделена на несколько уровней:
- **Unit Tests**: Проверка отдельных функций утилит из папки `System/Scripts/`.
- **Integration Tests**: Проверка взаимодействия между различными скриптами и агентами, например, корректность передачи данных от MNEMOSYNE к CASPER.
- **Stress Tests**: Сценарии для проверки поведения системы под высокой нагрузкой (например, при парсинге тысяч файлов одновременно).

## Правила использования и CI/CD
1. Ни один скрипт не может быть перенесен в `System/Scripts/` без соответствующего файла с тестами в `System/Tests/`.
2. VULCAN обязан автоматически запускать весь набор тестов при любом значительном изменении инфраструктуры.
3. Тесты должны быть идемпотентными и не оставлять после себя мусора в файловой системе (использовать моки и временные директории).

## Примеры артефактов
- `test_backup_script.py` - проверяет, корректно ли создается архив директории.
- `test_agent_routing.py` - симулирует сообщения между агентами и проверяет правильность выбора получателя.
- `mock_data_generator.py` - утилита для создания фиктивных файлов графа знаний для нужд тестирования.

Только благодаря строгому соблюдению тестового покрытия система может безопасно эволюционировать и масштабироваться.
