# 🐙 GITHUB SYNC

Скилл `github-sync` представляет собой критически важный инструмент автоматизации, обеспечивающий бесшовную двустороннюю синхронизацию локальной базы знаний и инфраструктуры "Карпаты" с удаленными Git-репозиториями. Это фундамент нашей системы контроля версий и CI/CD процессов.

### Архитектура и Интеграция

Инструмент реализован как защищенная обертка над стандартным CLI-клиентом Git и API GitHub. Он изолирует низкоуровневую логику работы с контролем версий, предоставляя агентам простой интерфейс.
- **Механизм Аутентификации:** Использует SSH-ключи или Personal Access Tokens (PAT), которые безопасно подгружаются из `Root_Configs`, исключая утечку кредов.
- **Автоматизированный Workflow:** Включает в себя логику `git add`, `git commit` с семантическими сообщениями (генерируемыми LLM), `git pull` с разрешением конфликтов и `git push`.
- **Интеграция с агентами:** Преимущественно используется агентом VULCAN (для деплоя кода и скриптов) и системными планировщиками (для создания ежедневных бэкапов).

### Принципы Работы и Использования

1. **Строгая ВаВалидация:** Любые деструктивные действия (такие как `force push` или удаление веток) либо заблокированы на уровне скилла, либо требуют явного подтверждения (Manual Approval) от Шефа.
2. **Семантическое Версионирование:** Скилл обучен генерировать осмысленные commit-сообщения, анализируя изменения в файлах (diff), что позволяет легко ориентироваться в истории изменений хранилища.
3. **Разрешение Конфликтов (Conflict Resolution):** При возникновении конфликтов слияния, инструмент не паникует, а формирует подробный отчет с маркерами конфликтов, передавая его обратно агенту для логического анализа и ручного/автоматического слияния.
4. **Аудит Трейлинг:** Каждая операция синхронизации оставляет лог, в котором фиксируется время, инициатор (имя агента) и хэш коммита.

С помощью `github-sync` мы гарантируем, что плоды работы ИИ-агентов надежно сохраняются, версионируются и могут быть развернуты на любой другой машине в считанные минуты.
