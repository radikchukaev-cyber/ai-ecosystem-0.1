# 📑 SKILLS REFERENCES

Директория `references` — это исчерпывающая база знаний, техническая документация и сборник мануалов по всем установленным инструментам (Tools) и скиллам (Skills) системы "Карпаты". Это внутренний "Stack Overflow" для наших ИИ-агентов.

### Архитектура Документации

Каждый скилл в системе обязан иметь свой reference-документ в этой директории. Качественная документация структурирована по жестким стандартам:
- **Спецификация API:** Точное описание принимаемых аргументов (Inputs), их типов (String, Boolean, Enum) и обязательности.
- **Формат Ответов (Outputs):** Структура данных, которые инструмент возвращает в случае успеха, а также примеры ответа.
- **Обработка Ошибок (Exceptions):** Список возможных кодов ошибок и рекомендации по их исправлению (Troubleshooting).
- **Примеры (Use Cases):** Готовые "копипаст"-фрагменты промптов и скриптов, показывающие, как правильно вызывать инструмент в реальных сценариях.

### Принципы Использования

1. **Self-Reflection для Агентов:** Перед тем как использовать незнакомый инструмент или написать новый скрипт интеграции, агенты (особенно VULCAN) обязаны прочитать соответствующий мануал из этой папки. Это снижает процент галлюцинаций LLM при вызове функций (Function Calling).
2. **Актуализация:** При любом обновлении исходного кода скилла, разработчик (или ответственный агент) обязан обновить его документацию. Устаревший reference считается критическим багом.
3. **Onboarding новых агентов:** При добавлении новой роли в систему, в ее стартовый промпт могут загружаться выдержки из `references`, чтобы агент сразу "понимал", какими возможностями он обладает.
4. **Единый Язык:** Документация пишется на понятном как для человека, так и для машины языке (Markdown), строго следуя семантической разметке.

Эта папка обеспечивает самообучаемость системы и снижает зависимость архитектуры от прямого микроменеджмента со стороны Шефа.
