# 🛡️ JSON PARSER SAFE

Скилл `json-parser-safe` — это утилита критической важности, предназначенная для обеспечения надежности межпрограммного взаимодействия (Machine-to-Machine). Она решает одну из главных проблем работы с LLM: склонность моделей добавлять "болтливость" (маркдаун, вступительные фразы, комментарии) вокруг структурированного вывода.

### Архитектура и Логика Работы

Данный инструмент выступает в качестве "санитайзера" и промежуточного слоя между сырым текстом ответа LLM и строго типизированными скриптами системы.
- **Эвристический поиск:** Скилл использует регулярные выражения и алгоритмы поиска сбалансированных скобок для сканирования всего текста и выявления потенциальных JSON-блоков (даже если они не обернуты в markdown-блоки ` ```json `).
- **Авто-восстановление (Auto-Healing):** Встроенные механизмы пытаются автоматически исправить типичные ошибки генерации: незакрытые кавычки, висячие запятые (trailing commas), неправильное экранирование спецсимволов.
- **Схема Валидации:** Интегрируется с JSON Schema. Извлеченный объект не просто парсится, но и проверяется на соответствие ожидаемой структуре перед передачей в пайплайн.

### Использование и Применение

1. **Повсеместная Интеграция:** Используется *абсолютно во всех* автоматизациях, где агент должен вернуть структурированные данные (например, генерация свойств Frontmatter, создание тасок, обновление статусов).
2. **Защита от Инъекций:** Парсер гарантирует, что исполняемый код не "сломается" от неожиданного текста, предотвращая цепочку отказов (cascading failures) в N8N-воркфлоу или скриптах VULCAN.
3. **Отладка:** Если JSON непоправимо поврежден, утилита выбрасывает структурированную ошибку с указанием номера строки и характера проблемы, что позволяет агенту-вызывающему сделать повторный (исправленный) запрос.
4. **Универсальность:** Поддерживает как парсинг простых объектов `{}`, так и массивов объектов `[]`, легко адаптируясь под различные форматы возвращаемых данных.

Без `json-parser-safe` автономная работа агентов была бы невозможна из-за постоянных сбоев при типизации потоковых данных.
