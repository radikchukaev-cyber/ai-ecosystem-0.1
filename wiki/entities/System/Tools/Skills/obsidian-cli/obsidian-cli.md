# 📓 OBSIDIAN CLI

Скилл `obsidian-cli` представляет собой мощный интерфейс командной строки (CLI) и набор API-оберток для программного управления локальным графом заметок. Он перекидывает мост между миром скриптов (терминалом) и богатой экосистемой Obsidian, позволяя агентам управлять хранилищем "изнутри".

### Архитектура Инструмента

Инструмент напрямую взаимодействует с файловой системой и локальными плагинами (такими как Advanced URI или Local REST API), обеспечивая глубокую интеграцию.
- **Управление Заметками:** CRUD-операции (создание, чтение, обновление, удаление) над Markdown-файлами с сохранением целостности графа и ссылок.
- **Работа с Метаданными:** Парсинг и модификация YAML Frontmatter без нарушения структуры документа (с использованием AST или безопасных regex).
- **Управление Интерфейсом:** Возможность программно открывать файлы, запускать команды Obsidian (Command Palette), переключать рабочие пространства (Workspaces) и активировать плагины.

### Использование и Сценарии

1. **Фундамент МНЕМОЗИНЫ:** Для агента MNEMOSYNE (Хранитель Памяти) этот скилл является основным "рабочим органом". Через него она перестраивает индексы, актуализирует теги и связывает изолированные заметки (orphan notes) в смысловые кластеры.
2. **Массовый Рефакторинг:** VULCAN использует `obsidian-cli` для переименования директорий и автоматической замены путей во всех связанных файлах, предотвращая появление "битых" ссылок.
3. **Ежедневные Рутины:** Инструмент автоматически генерирует ежедневные (Daily) и еженедельные (Weekly) заметки на основе шаблонов (Templater), предзаполняя их погодными данными, календарем и тасками.
4. **Асинхронность:** Скилл поддерживает работу в фоне, позволяя агентам индексировать тысячи файлов, не подвешивая графический интерфейс редактора Шефа.

По сути, `obsidian-cli` превращает Obsidian из статического блокнота в динамическую, управляемую базу данных.
