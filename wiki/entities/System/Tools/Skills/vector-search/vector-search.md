# 🔍 VECTOR SEARCH SKILL

Скилл `vector-search` — это ядро системы поиска и извлечения знаний (Retrieval-Augmented Generation, RAG). Этот инструмент наделяет агентов способностью "понимать" контекст базы знаний "Карпаты", выполняя поиск не по строгому совпадению ключевых слов, а по смысловой близости (семантике).

### Архитектура и Технологический Стек

Инструмент построен на базе современных технологий векторизации и хранения данных:
- **Embedding Models:** Входящие тексты и запросы пропускаются через специализированные эмбеддинг-модели (например, OpenAI text-embedding-3 или локальные аналоги), превращаясь в многомерные числовые вектора.
- **Векторная БД (Vector Store):** Использование локальной или облачной векторной базы данных (ChromaDB, Pinecone, Qdrant) для быстрого вычисления косинусного сходства (Cosine Similarity) между запросом и тысячами заметок.
- **Чанкинг (Chunking):** Алгоритмы умной разбивки длинных Markdown-документов на семантические блоки (абзацы, заголовки) для обеспечения точного поиска контекста, а не просто всей заметки.

### Использование и Применение

1. **Контекстуализация промптов:** Когда Шеф задает агенту вопрос, `vector-search` первым делом сканирует базу знаний, извлекает релевантные куски текста и подмешивает их в системный промпт (Context Injection). Это исключает галлюцинации и опирается только на реальные данные.
2. **Связывание (Graph Building):** Агент MNEMOSYNE использует этот скилл в фоновом режиме, чтобы находить неочевидные связи между заметками (например, соединяя концепцию из книги по психологии с рабочим проектом) и предлагать новые wikilink'и.
3. **Дедупликация:** При добавлении новой информации инструмент проверяет базу на наличие "похожих по смыслу" идей, предлагая Шефу дополнить существующую заметку, а не плодить дубликаты.
4. **Гибридный поиск:** Часто комбинируется с точным лексическим поиском (BM25 или grep), чтобы объединить мощь понимания смысла и точность поиска конкретных имен или терминов.

`vector-search` превращает хранилище из простого архива в активного собеседника и помощника.
