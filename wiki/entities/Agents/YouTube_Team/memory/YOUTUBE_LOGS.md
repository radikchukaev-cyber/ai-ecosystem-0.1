# Журнал: YOUTUBE_LOGS

## Обзор
Централизованный журнал операций, логов и событий, связанных с работой агентов YouTube-подразделения (Creator, Editor, SEO, Thumbnail Artist, Analytics).

## Назначение
Этот лог используется для отслеживания статусов выполнения задач, записи ошибок взаимодействия между агентами и сохранения истории публикаций для последующего анализа.

## Структура Записей
Каждая запись в логе должна содержать:
- `Timestamp`: Точное время события (ISO 8601).
- `Agent`: Имя агента, инициировавшего запись (например, SEO_Agent).
- `Level`: Уровень важности (INFO, WARNING, ERROR, CRITICAL).
- `Action`: Краткое описание действия (например, 'Thumbnail generated', 'Video uploaded').
- `Details`: JSON объект с дополнительными параметрами (ID видео, метрики, пути к файлам).

## Политика Хранения
- Горячие логи (последние 30 дней) хранятся в оперативной памяти/быстрой БД для мгновенного доступа аналитиков.
- Холодные логи (старше 30 дней) архивируются в сжатые форматы и отправляются в долгосрочное хранилище.

## Мониторинг
Агенты-аудиторы регулярно сканируют этот журнал на предмет аномалий (например, слишком много записей ERROR от Editor Agent при рендере видео).
