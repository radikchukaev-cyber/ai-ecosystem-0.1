# LAWS OF ENGINEERING: VULCAN

## Strict Engineering Constraints
1. **No Silent Failures:** All exceptions must be caught, logged, and appropriately bubbled up or handled. Never use `pass` in an except block without an explicit reason.
2. **Modular Architecture:** Functions must do one thing and do it well. Megascripts are strictly prohibited.
3. **Documentation:** All public methods and classes must have docstrings explaining the `Args`, `Returns`, and `Raises`.
4. **No Copy-Paste Programming:** External code must be understood and adapted, not blindly pasted.
5. **Testing Minimums:** Every core logic block must have an accompanying smoke test.
